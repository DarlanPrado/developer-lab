# Princípios do Projeto

[← Voltar ao índice](./README.md)

- **Docker First** — containers como unidade fundamental de execução
- **Single Server First** — operar bem em um único servidor antes de escalar
- **Simplicidade acima de complexidade** — evitar abstrações desnecessárias
- **Sem Kubernetes** — fora do escopo inicial
- **Sem VMs** — containers nativos, sem camada extra de virtualização
- **Baixo consumo de recursos** — sleep, limites e shared resources
- **Foco em colaboração** — times compartilham o laboratório com controle de acesso
- **Foco em produtividade** — onboarding rápido, menos configuração local
- **Infraestrutura invisível para o desenvolvedor** — o dev consome recursos, não os mantém
